## 核心数据结构

```c
struct ngx_event_s {
  void *data;
  ngx_event_handler_pt handler;
  // 红黑树
  ngx_rbtree_node_t timer;
  ngx_queue_t queue;
}

struct ngx_connection_s {
  // 读写时间
  ngx_event_t *read;
  ngx_event_t *write;
  // 抽象解耦OS底层方法
  ngx_recv_pt recv;
  ngx_send_pt send;
  // bytes_sent变量
  off_t sent;
  // 初始connection_poll_size配置
  ngx_poll_t *poll;
}
```

## 进程间的通讯方式

1. 基础同步工具

- 信号
- 共享内存  
  跨 worker 进程通讯  
  共享内存使用者:
  - rbtree(红黑树): ngx_stream_limit_conn_module,ngx_http_limit_conn_module, ngx-stream_limit_req_module,http cache,ssl
  - 单链表: ngx_http_upstream_zone_module,ngx_stream_upstream_zone_module

2. 高级通讯方式

- 锁
- slab 内存管理器 (slab_state)
  bestfit:
  - 最多两倍内存消耗
  - 适合小对象
  - 避免碎片
  - 避免重复初始化

## 配置嵌套

指令的 Context

1. 值指令: 存储配置项的值

- 可以合并
- 实例:
- root
- access_log
- gzip

2. 动作指令: 指定行为

- 不可以合并
- 示例:
  - rewrite
  - proxy_pass
- 生效阶段
  - server_rewrite 阶段
  - rewrite
  - context

## 存储值的指令:向上覆盖

子配置不存在时,直接使用父配置块
子配置存在时,直接覆盖父配置块

```conf
server {
  listen 8080;
  root /home/nginx;
  access_log logs/access.log main;
  location /test {
    root /home/nginx/test;
    access_log logs/access.test.log main;
  }
}
```
